# 第9章：故障排查与日志分析

> 本章提供 OpenClaw 故障排查的系统方法，包括诊断工具、日志分析和常见问题解决方案。

---

## 9.1 诊断工具

### openclaw doctor

```bash
openclaw doctor        # 全面诊断
openclaw doctor --fix  # 自动修复
```

检查项目：
- Node.js 版本
- Gateway 服务状态
- 配置文件格式
- Skills 加载状态
- 通道连接状态
- 磁盘空间

---

## 9.2 日志系统

### 查看日志

```bash
openclaw logs --follow         # 实时日志
journalctl --user -u openclaw-gateway -f  # systemd 日志
```

### 日志级别

| 级别 | 说明 |
|------|------|
| DEBUG | 详细调试信息 |
| INFO | 常规运行信息 |
| WARN | 警告但不影响运行 |
| ERROR | 错误，需要关注 |
| FATAL | 致命错误，服务终止 |

---

## 9.3 常见故障与解决

### 1: Gateway 无法启动

```bash
# 检查端口
lsof -i :18789
# 检查配置
cat ~/.openclaw/openclaw.json | python3 -m json.tool
# 重新安装服务
openclaw daemon install --force
```

### 2: Agent 无响应

```bash
# 检查 Gateway 连接
openclaw gateway status --deep
# 检查模型配置
# 确认 API Key 有效
```

### 3: Skill 加载失败

```bash
# 验证 SKILL.md 格式
head -20 ~/.openclaw/workspace/skills/<name>/SKILL.md
# 检查依赖
openclaw doctor
```

## 常见问题

| 问题 | 解决方法 |
|------|---------|
| 日志在哪里 | `~/.openclaw/logs/` 和 `journalctl --user -u openclaw-gateway` |
| 如何重置配置 | 备份后删除 `~/.openclaw/openclaw.json`，运行 `openclaw onboard` 重新初始化 |
